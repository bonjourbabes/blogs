C is a mid-level computer language developed in the early 1970s to do programming for UNIX and has later influenced many other computer languages such as Java and C++. Now, in order to use C files, they have to go through a compilation process (gcc) to create a file that can be executed. The source code is not comprehensible and needs to be converted into object code. Object code or machine code looks like computer language with a lot of binary code. The compiler makes sure that the source code is error-free. After doing that, the compiler will generate the object code. Imagine an archaeologist finds a Now in the following steps, I will instruct the proper steps to compile a file in C Step 1 ) Open up the terminal, type either vi or emacs then filename.c Here are some examples:

$ emacs filename.c $ vi main.c $ emacs file.c

It is very important that the file’s name ends in c.

Step 2) In the terminal type this: $ gcc filename.c -o filename

GCC stands for “GNU Compiler Collection”. The -o flag is used to create a specific name for the output file. If the -o option is not used, then an output file named a.out will be generated instead.

Step 3) The executable file is generated. Now run the executable file by doing this: $ ./file

Now, let’s dive into the compilation process and have a deeper understanding of how the compiler conversion occurs so that a C program becomes an executable file. The process can be divided into four phases: Pre-processing Compilation Assembly Linking The first phase is the preprocessor. It receives the source code, removes all the comments from the source code, looks at the source code for special instructions called Preprocessor directives, like . If the aforementioned directive is present, the preprocessor interprets the directive and then puts in the content from the . Basically, the preprocessor looks at the file and gets told what to do by the preprocessor directives. If there is an arithmetic question or program, it is more than likely that will need to be used at the top of the file. Or else the program will be unable to run.

These directives are recognized by the preprocessor: #define #elif #else #endif #error #if #ifdef #ifndef #import #include #line #pragma #undef #using

The second phase is Compilation. After the expanded code by the preprocessor goes to the compiler, what does this mean? The pre-processed code now goes into assembly code. For example, a file called file.c is run through the preprocessor and then becomes file.i after being expanded. The file.i is simply an extension of the expanded source code. The third phase is the Assembler. Now that we have our assembly code, it will be converted into object code by using an assembler. If you try to open up an assembly code file, it will be be full of characters that have no meaning to you. These files can end with .o. The fourth phase is the Linker. This is a very important phase because the Linker knows where all these functions are implemented, it puts all the object files and makes them into a file. The executable file ends with .exe. Do you have to compile every c file whenever you want to execute it? Yes, you do. The process cannot be changed drastically, but it is doable.
